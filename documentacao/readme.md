# 🔄 Retrospectiva do Projeto

Este documento apresenta a retrospectiva do desenvolvimento da plataforma web para análise de aspectos Ambientais, Sociais e de Governança (ASG) de propriedades rurais do Estado de São Paulo.

---

## ✅ Pontos Fortes

### Clareza do objetivo do projeto
O grupo compreendeu bem a proposta de desenvolver uma plataforma voltada à análise ASG, com foco em propriedades rurais do estado de São Paulo.

### Estrutura inicial bem definida
Houve uma boa organização no início do projeto, com definição clara de funcionalidades como:
- Visualização da propriedade no mapa  
- Análise de dados ambientais  
- Integração com fontes externas  

### Uso de tecnologias adequadas
Foram utilizadas tecnologias coerentes com o cenário proposto, como:
- ETL para coleta de dados  
- Bancos de dados  
- APIs externas  

### Divisão inicial de tarefas
No início, houve uma tentativa de dividir as responsabilidades entre os integrantes, o que ajudou no avanço das primeiras etapas.

### Evolução técnica individual
Mesmo com dificuldades no trabalho em grupo, houve aprendizado individual relevante, principalmente em temas como cloud, integração de dados e desenvolvimento.

---

## ❌ Pontos Fracos

### Distribuição de tarefas desigual
Alguns integrantes ficaram responsáveis por grande parte do projeto, enquanto outros contribuíram pouco.

### Falta de organização contínua
A organização inicial não foi mantida ao longo do projeto, faltando acompanhamento e controle das atividades.

### Comunicação ineficiente
Ocorreram falhas na comunicação, como:
- Atrasos nas respostas  
- Falta de alinhamento  
- Decisões não compartilhadas  

### Pouco acompanhamento das entregas
Não houve controle efetivo dos prazos e das entregas individuais, resultando em atrasos e retrabalho.

---

## 🔧 Ações de Melhoria

Com base nos pontos fracos identificados, foram definidas as seguintes ações para melhoria em projetos futuros:

### Organização e gestão (Jira)
- Definir responsáveis claros para cada tarefa no Jira  
- Estabelecer prazos individuais para cada atividade  
- Atualizar o status das tarefas com frequência (To Do / In Progress / Done)  

### Qualidade e controle
- Padronizar a qualidade das entregas 
- Realizar revisões periódicas do projeto  

### Comunicação e colaboração
- Estabelecer tempo máximo para resposta entre os membros  
- Registrar decisões importantes para evitar desalinhamento  

### Acompanhamento e melhoria contínua
- Realizar checkpoints semanais para acompanhar o progresso  
- Identificar rapidamente atrasos e redistribuir tarefas quando necessário  

---

## 📌 Considerações Finais

Apesar das dificuldades enfrentadas durante o desenvolvimento, o projeto proporcionou aprendizado significativo, tanto técnico quanto em relação ao trabalho em equipe.
